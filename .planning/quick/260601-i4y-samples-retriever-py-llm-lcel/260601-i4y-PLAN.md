Create 6 new example scripts in examples/ that demonstrate retriever-only usage
(no chain, no LLM, no create_llm import). Each file follows the same structure
as the existing *_query.py examples but stops after retriever.ainvoke().

UNIVERSAL PATTERN for each file:
1. Docstring explaining the mode and usage (Chinese + English technical terms).
   Include: `uv run python examples/X_retriever.py`
2. `sys.path.insert(0, str(Path(__file__).parent.parent))` for project root
3. Imports: the retriever class, `settings` from config, `init_pool` from pool,
   `PGVectorStore` from store. For modes needing graph: also `PGGraphStore` from graph.
   Do NOT import: create_llm, any Chain class, any LLM class.
4. `async def main()`:
   a. `await init_pool()` + create stores (PGVectorStore, PGGraphStore as needed per mode)
   b. Create retriever with correct args (see constructor signatures in context)
   c. Define a query string
   d. `docs = await retriever.ainvoke(query)` — returns list[Document]
   e. Print mode, query, and document count
   f. For each doc: print `page_content` (first 200 chars + "..." if longer)
      and all `metadata` keys/values
   g. If no documents found: print a clear "no results" message
5. `if __name__ == "__main__": asyncio.run(main())`

MODE-SPECIFIC DETAILS:

naive_retriever.py — Naive mode: vector chunk search only
- Imports: NaiveRetriever, settings, init_pool, PGVectorStore
- Stores: vector_store = PGVectorStore() (NO graph_store)
- Retriever: NaiveRetriever(vector_store=vector_store, embedding_config=settings.embedding)
- Query: "启动东莞市防风Ⅰ级应急响应后需要落实哪些措施？" (same as naive_query.py)
- Expected: list of chunk Documents

local_retriever.py — Local mode: entity vector search + graph expansion
- Imports: LocalRetriever, settings, init_pool, PGVectorStore, PGGraphStore
- Stores: vector_store = PGVectorStore(), graph_store = PGGraphStore()
- Retriever: LocalRetriever(vector_store=vector_store, graph_store=graph_store, embedding_config=settings.embedding)
- Query: "广州市三防成员单位有哪些？" (same as mix_query.py)
- Expected: entity + GraphTriple Documents

global_retriever.py — Global mode: relation vector search + entity lookup
- Imports: GlobalRetriever, settings, init_pool, PGVectorStore, PGGraphStore
- Stores: vector_store = PGVectorStore(), graph_store = PGGraphStore()
- Retriever: GlobalRetriever(vector_store=vector_store, graph_store=graph_store, embedding_config=settings.embedding)
- Query: "广州市三防成员单位有哪些？"
- Expected: relation + GraphTriple Documents

hybrid_retriever.py — Hybrid mode: parallel local + global
- Imports: HybridRetriever, settings, init_pool, PGVectorStore, PGGraphStore
- Stores: vector_store = PGVectorStore(), graph_store = PGGraphStore()
- Retriever: HybridRetriever(vector_store=vector_store, graph_store=graph_store, embedding_config=settings.embedding)
- Query: "广州市三防成员单位有哪些？"
- Expected: entity + relation + GraphTriple Documents

mix_retriever.py — Mix mode: hybrid retrieval + chunk vector search
- Imports: MixRetriever, settings, init_pool, PGVectorStore, PGGraphStore
- Stores: vector_store = PGVectorStore(), graph_store = PGGraphStore()
- Retriever: MixRetriever(vector_store=vector_store, graph_store=graph_store, embedding_config=settings.embedding)
- Query: "广州市三防成员单位有哪些？"
- Expected: entity + relation + chunk + GraphTriple Documents

bypass_retriever.py — Bypass mode: always returns empty list
- Imports: BypassRetriever, settings, init_pool, PGVectorStore
- Stores: vector_store = PGVectorStore() (NO graph_store — BypassRetriever does not use it)
- Retriever: BypassRetriever(vector_store=vector_store, embedding_config=settings.embedding)
- Query: "请简要介绍中国的应急管理体系。" (same as bypass_query.py)
- Expected: empty list [] — print "Bypass mode: no documents retrieved (expected)"
- NOTE: BypassRetriever does not use vector_store or embedding_config at runtime
  (returns [] without any I/O), but both are required constructor args because the
  base class Pydantic model requires them. init_pool() and PGVectorStore() are still
  called for consistent setup pattern.
