# Milestone v1.0: lightrag-langchain 初始发布

**Status:** ✅ SHIPPED 2026-06-01
**Phases:** 1-7
**Total Plans:** 23

## Overview

基于 Langchain 框架的 LightRAG 查询层，直接读取 LightRAG 已处理好的 PostgreSQL 知识图谱数据库，复刻全部六种查询模式（naive / local / global / hybrid / mix / bypass），提供标准 Langchain Retriever + Chain 接口。

## Phases

### Phase 1: Configuration

**Goal**: .env-based configuration for all services (PG, LLM, Embedding, Reranker, QueryParams)
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: Project scaffolding: pyproject.toml, package structure, .gitignore, .env.example, test fixtures
- [x] 01-02: Configuration implementation: 5 sub-models, Settings singleton, fail-fast validation, 30 tests (TDD)

### Phase 2: Data Layer

**Goal**: Read-only PostgreSQL access layer for LightRAG's PGVector and Apache AGE graph stores
**Depends on**: Phase 1
**Plans**: 4 plans

Plans:
- [x] 02-01: Data models: EntityRecord, RelationshipRecord, ChunkRecord, GraphNode, GraphEdge (Pydantic frozen)
- [x] 02-02: Config extension (workspace, pool params) + Connection pool manager (asyncpg lazy init, retry, read-only)
- [x] 02-03: PGVectorStore: entities/relationships/chunks vector similarity search with table auto-discovery
- [x] 02-04: PGGraphStore: AGE graph node/edge query, batch lookup, and neighbor traversal

### Phase 3: LLM Integration

**Goal**: LLM, embedding, and reranker services integrated with provider-agnostic interfaces and token budget enforcement
**Depends on**: Phase 1
**Plans**: 5 plans

Plans:
- [x] 03-01: Config extension (pyproject.toml deps, KEYWORD_LANGUAGE, conftest fixtures)
- [x] 03-02: LLM factory: create_llm() + create_embedding() lazy proxies
- [x] 03-03: Reranker Integration: Reranker Protocol, 3 adapters, create_reranker(), LightRAGReranker
- [x] 03-04: Token Budget: truncate functions + chunk budget calculator
- [x] 03-05: Keyword Extraction + Module Export: KeywordsSchema, extract_keywords(), __init__.py lazy exports

### Phase 4: Query Strategies

**Goal**: All 6 LightRAG query modes produce correct, distinct retrieval results
**Depends on**: Phase 2, Phase 3
