    Per D-04 relationship JSON format (fields match upstream lines 3228-3238):
    ```json
    {"src_id": relation.src_id, "tgt_id": relation.tgt_id, "description": relation.content or "", "keywords": keywords, "weight": weight, "source_id": source_id, "file_path": file_path}
    ```

    - file_path from caller — empty string if unavailable (neither VDB nor AGE stores file_path for relations)

    ```python
    metadata = {
        "source_id": source_id or "",

        "document_type": "relation",
        "src_id": relation.src_id,
        "tgt_id": relation.tgt_id,
        "keywords": keywords or relation.keywords or "",
        "weight": weight if weight is not None else relation.weight,
    }
    ```

    Weight handling: if neither passed nor on record, omit (set to None). The JSON `json.dumps` will produce `null`.

    **Function 5: `build_graph_lookups(triples: list[GraphTriple]) -> tuple[dict[str, GraphNode], dict[tuple[str, str], GraphEdge]]`**

    - Deduplication: if multiple triples have same entity_id, last one wins (all triples for same entity carry the same node data).
