## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user query → extract_keywords() | Untrusted user query string enters keyword extraction prompt |
| extract_keywords() → LLM | Prompt (including user query) sent to external LLM API |
| LLM response → extract_keywords() | LLM output deserialized into KeywordsSchema Pydantic model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Tampering | Prompt injection via user query | mitigate | Structured output mode constrains LLM to JSON schema output. The upstream prompt template includes role constraints (expert system, instructions). Pydantic KeywordsSchema validates the response shape — non-conforming responses are rejected by with_structured_output. |
| T-03-05-02 | Information Disclosure | Template strings in source | accept | Prompt templates are public, non-sensitive strings from open-source LightRAG. No secrets embedded. |
| T-03-05-03 | Elevation of Privilege | __init__.py lazy imports | accept | Lazy import pattern is standard Python (used by unittest.mock, sqlalchemy, etc.). No privilege escalation risk — it's a module loading mechanism. |
| T-03-05-04 | Spoofing | LLM hallucinated keywords | accept | By design, LLM may produce hallucinated keywords. This is inherent to LLM-based extraction. Downstream retrieval is best-effort. |
| T-03-05-SC | Tampering | pydantic dependency | mitigate | Already in project dependencies (Phase 1). Passed previous phases' audits. |
</threat_model>
