## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller -> strategy functions | Untrusted `query_embedding: list[float]` crosses boundary |
| Strategy functions -> PGVectorStore | Embedding vector passed to parameterized SQL -- pgvector rejects malformed vectors |
| Strategy functions -> PGGraphStore | Entity names from VDB used as graph node IDs -- names controlled at DB level |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | query_embedding input | mitigate | Type hint `list[float]` enforced by Python; PGVectorStore._vector_search uses `$4::vector` parameterized binding -- pgvector rejects malformed vectors at SQL level |
| T-04-02 | Denial of Service | local_strategy graph traversal | mitigate | Bounded by top_K parameter (default 40); asyncio.gather with return_exceptions=True prevents one failed lookup from crashing the batch; PGGraphStore handles empty results gracefully |
| T-04-03 | Information Disclosure | global_strategy error paths | mitigate | Return empty QueryResult() on errors (not stack traces); logger.warning captures entity IDs for debugging but not full content; no raise in production paths |
| T-04-SC | Tampering | No new npm/pip/cargo packages | accept | Zero new dependencies -- only stdlib (asyncio, logging) + existing project deps (pydantic, asyncpg) |
