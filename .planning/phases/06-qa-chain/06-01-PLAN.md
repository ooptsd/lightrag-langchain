Define 4 functions. Each function parses `doc.page_content` (JSON string) via `json.loads()` and returns a dict. All use `.get(key, default)` for safe key access. Function signatures and return dicts mirror RESEARCH.md lines 401-447:

**1. doc_to_entity_dict(doc: Document) -> dict**
Parse entity Document. Return dict with string keys: `entity_name`, `entity_type`, `description`, `source_id`, `file_path`. Fallback for all: `""`.
Source: retriever/utils.py entity_to_document line 66-72 shows the JSON shape.

**2. doc_to_relation_dict(doc: Document) -> dict**
Parse relation Document. Return dict with keys: `src_id` (str), `tgt_id` (str), `description` (str), `keywords` (str), `weight` (float), `source_id` (str), `file_path` (str). Fallback: `weight` defaults to `0.0`, others to `""`.
Source: retriever/utils.py relation_to_document line 133-140 shows the JSON shape.

**3. doc_to_chunk_dict(doc: Document) -> dict**
Parse chunk Document. Return dict with keys: `content` (str), `file_path` (str), `chunk_id` (str), `reference_id` (str). Fallback: all default to `""`. The `reference_id` field starts as empty string — Plan 06-02's `_build_reference_list()` fills it in later.
Source: retriever/utils.py chunk_to_document line 185-189 shows the JSON shape.

**4. classify_and_convert(docs: list[Document]) -> tuple[list[dict], list[dict], list[dict]]**
Iterate all Documents, dispatch by `doc.metadata.get("document_type", "")`.
- `"entity"` → call doc_to_entity_dict, append to entities list
- `"relation"` → call doc_to_relation_dict, append to relations list
- `"chunk"` → call doc_to_chunk_dict, append to chunks list
- `"graph_triple"` → SKIP (no conversion — RESEARCH.md lines 449-454: GraphTriple data already captured through entity/relation Documents, file_path is always "")
- Unknown type → skip silently
Return `(entities, relations, chunks)` tuple.

All 4 functions are pure — no class, no global state, no logging, no async. Exceptions from `json.JSONDecodeError` propagate (Claude's Discretion: chain layer does no error recovery).
