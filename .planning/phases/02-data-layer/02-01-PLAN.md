Model 1 — EntityRecord (maps to LIGHTRAG_VDB_ENTITY table):
- entity_name: str — the entity name (VDB_ENTITY.entity_name column, VARCHAR(512))
- content: str — entity textual content (VDB_ENTITY.content column)
- source_id: str — entity unique identifier (VDB_ENTITY.id column, renamed per STOR-01)
- file_path: str = "" — source file path, COALESCE'd from NULL (VDB_ENTITY.file_path column)
- created_at: int | None = None — epoch seconds from create_time (EXTRACT(EPOCH FROM create_time)::BIGINT)

Model 2 — RelationshipRecord (maps to LIGHTRAG_VDB_RELATION table):
- src_id: str — source entity ID (VDB_RELATION.source_id column)
- tgt_id: str — target entity ID (VDB_RELATION.target_id column)
- content: str | None = None — relation textual content (VDB_RELATION.content column)
- keywords: str | None = None — relation keywords. Default None because VDB_RELATION DDL does NOT contain keywords/weight columns (per RESEARCH.md Open Question 1 and A1 assumption). Real values come from AGE graph edges (Plan 02-04). PGVector search returns NULL for these.
- weight: float | None = None — relation weight. Same as keywords: NULL from VDB_RELATION, real value from AGE graph edges.
- created_at: int | None = None — epoch seconds from create_time

Model 3 — ChunkRecord (maps to LIGHTRAG_VDB_CHUNKS table):
- chunk_id: str — chunk identifier (VDB_CHUNKS.id column)
- content: str — chunk textual content (VDB_CHUNKS.content column)
- full_doc_id: str | None = None — parent document identifier (VDB_CHUNKS.full_doc_id column)
- chunk_order_index: int | None = None — position in document sequence (VDB_CHUNKS.chunk_order_index column)
- file_path: str = "" — source file path, COALESCE'd from NULL (VDB_CHUNKS.file_path column)

Model 4 — GraphNode (maps to AGE graph node under label "base"):
- entity_id: str — node identity, from properties.entity_id on AGE node
- entity_type: str — node type, from properties.entity_type on AGE node
- description: str = "" — entity description, from properties.description on AGE node (may be NULL in DB)
- source_id: str = "" — document source, from properties.source_id on AGE node (may be NULL in DB)

Model 5 — GraphEdge (maps to AGE graph edge under label "DIRECTED"):
- source_id: str — source entity ID (src node's entity_id)
- target_id: str — target entity ID (tgt node's entity_id)
- description: str | None = None — edge description, from properties.description on AGE edge
- keywords: str | None = None — edge keywords, from properties.keywords on AGE edge
- weight: float | None = None — edge weight, from properties.weight on AGE edge

Do NOT import asyncpg or any database library — models.py is pure Pydantic with zero runtime dependencies beyond pydantic (already in pyproject.toml from Phase 1).
