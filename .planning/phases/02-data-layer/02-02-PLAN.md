Define async init_pool(custom_pool: Pool | None = None) -> Pool:
- If custom_pool is not None: set _pool = custom_pool, return _pool. This is the dependency injection path (D-07) — callers can pass a shared pool from outside.
- If _pool is already set (not None): return _pool immediately. Idempotent.
- Otherwise: create a new pool via asyncpg.create_pool() with explicit keyword arguments:
  - host=settings.pg.host
  - port=settings.pg.port
  - user=settings.pg.user
  - password=settings.pg.password.get_secret_value()  # SecretStr unwrap
  - database=settings.pg.database

  - init=_init_connection  # callback for each new connection
  - server_settings={'application_name': 'lightrag_langchain', 'default_transaction_read_only': 'on'}  # D-15 database-level read-only
- Return the created pool.

Define async _init_connection(conn: asyncpg.Connection) -> None:
- Calls await register_vector(conn) to register pgvector binary codec on each pool connection
- Import: from pgvector.asyncpg import register_vector

Define async close_pool() -> None:
- If _pool is not None: await _pool.close(), then set _pool = None
- If _pool is already None: no-op (idempotent close)

Define async acquire_with_retry(pool: Pool, max_retries: int = 3) -> AsyncIterator[Connection]:
- Async generator that yields an acquired connection
- On transient errors during pool.acquire() (asyncpg.exceptions.ConnectionDoesNotExistError, asyncpg.exceptions.ConnectionFailureError, OSError, TimeoutError), retries with exponential backoff: 1s, 2s, 4s (D-06: 3 retries max)
- On non-transient errors (e.g., syntax errors, query logic errors): re-raise immediately without retry
- On final retry exhaustion: re-raise the last exception
- Use asyncio.sleep(n) for backoff delay
- Implementation: try pool.acquire(), if exception matches transient types and retries remaining > 0, sleep, decrement retries, retry. After loop, if still inside async with equivalent context, yield conn; in finally, release conn back to pool. Use try/finally for safe connection release.

All functions are async (needed for asyncpg). Use import asyncpg at module level (not TYPE_CHECKING) since pool creation calls asyncpg.create_pool(). Import settings: from lightrag_langchain.config import settings. Import register_vector: from pgvector.asyncpg import register_vector.

Do NOT create a PGVectorStore or PGGraphStore class — those are separate plans (02-03, 02-04). Do NOT add table discovery logic. Do NOT add query methods. pool.py manages only the connection pool lifecycle.
