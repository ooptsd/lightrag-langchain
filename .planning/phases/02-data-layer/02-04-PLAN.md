— Helper: _dollar_quote(s: str, tag_prefix: str = "AGE") -> str:
From LightRAG pattern (L:112-140). Generates dollar-quoted string with unique tag to avoid conflict with Cypher content containing $$. Iterates itertools.count(1), generates tag = f"{tag_prefix}{i}", wrapper = f"${tag}$". If wrapper not in s, returns f"{wrapper}{s}{wrapper}". If s contains every generated wrapper up to a reasonable limit, fall back to a random tag.

— Helper: _parse_agtype(value: str) -> dict | None:
Parses AGE agtype return strings which may have "::vertex" or "::edge" type suffixes. Split by "::" — take everything before the last "::" as JSON content. If value is empty or whitespace, return None. Try json.loads(content). On JSONDecodeError, return None (graceful degradation for edge cases).

— Method async _resolve_graph_name(self) -> str:
If self._graph_name_resolved is not None: return it.
Query AGE metadata: SELECT ag_catalog.ag_graph_exists($1) or query information_schema for schemas matching ag_catalog pattern. Simplified approach: use settings.pg.workspace to derive graph name.
  - If workspace is "default" or empty: graph name = "lightrag_graph" (standard LightRAG default)
  - Otherwise: graph name = re.sub(r"[^a-zA-Z0-9_]", "_", workspace) + "_lightrag_graph" (D-05 sanitization)
Store result in self._graph_name_resolved and return.
If .env PG_GRAPH_NAME is provided (check settings.pg for graph_name field, or default), use that as override (D-14). For now, derive from workspace as above.

— Method async _query(self, cypher: str, params: dict, returns: str) -> list[dict]:
Core AGE query wrapper. Uses SELECT * FROM cypher(graph_name, cypher_query, $1::agtype) AS ({returns}) pattern.
- Build graph_name_quoted = self._dollar_quote(self._graph_name_resolved)
- Build cypher_quoted = self._dollar_quote(cypher)
- Build full SQL: f"SELECT * FROM cypher({graph_name_quoted}::name, {cypher_quoted}::cstring, $1::agtype) AS ({returns})"
- Serialize params: pg_params = {"params": json.dumps(params, ensure_ascii=False)} (Pitfall 3 from RESEARCH.md)
- async with acquire_with_retry(self.pool) as conn: rows = await conn.fetch(sql, pg_params["params"])
- Return [dict(row) for row in rows]

— Method async get_node(self, node_id: str) -> GraphNode | None:
Cypher: "MATCH (n:base {entity_id: $entity_id}) RETURN properties(n) AS props"
Returns: "props agtype"

Construct GraphNode(entity_id=node_id, entity_type=props.get("entity_type", ""), description=props.get("description", ""), source_id=props.get("source_id", "")).

— Method async get_nodes_batch(self, node_ids: list[str]) -> dict[str, GraphNode]:
If node_ids is empty: return {}.
Uses UNNEST + ag_catalog.agtype_access_operator pattern from LightRAG (L:5353-5431) for parameterized batch lookup. Build SQL directly (bypass _query helper since this uses a different structure):
```sql
WITH input(v, ord) AS (
  SELECT v, ord
  FROM unnest($1::text[]) WITH ORDINALITY AS t(v, ord)
),
ids(node_id, ord) AS (
  SELECT (to_json(v)::text)::agtype AS node_id, ord
  FROM input
)
SELECT i.node_id::text AS node_id,
       b.properties
FROM {graph_name}.base AS b
JOIN ids i
  ON ag_catalog.agtype_access_operator(
       VARIADIC ARRAY[b.properties, '"entity_id"'::agtype]
     ) = i.node_id
ORDER BY i.ord
```
Execute with conn.fetch(sql, node_ids). For each row: parse node_id (strip quotes if agtype text), parse properties via _parse_agtype, construct GraphNode. Return dict mapping node_id -> GraphNode.

— Method async get_edge(self, src: str, tgt: str) -> GraphEdge | None:
Cypher: "MATCH (a:base {entity_id: $src})-[r:DIRECTED]->(b:base {entity_id: $tgt}) RETURN properties(r) AS props"
Returns: "props agtype"
Call self._query(cypher, {"src": src, "tgt": tgt}, returns). If rows empty: return None. Parse props from row["props"]. Construct GraphEdge(source_id=src, target_id=tgt, description=props.get("description"), keywords=props.get("keywords"), weight=props.get("weight") — all may be None).

— Method async get_edges_batch(self, pairs: list[dict[str, str]]) -> dict[tuple[str, str], GraphEdge]:
If pairs is empty: return {}.
Split into forward query and backward query patterns, or use UNWIND. For simplicity: iterate pairs and call get_edge for each. For large batches (length > 10), use UNWIND Cypher:
```
UNWIND $pairs AS pair
MATCH (a:base {entity_id: pair.src})-[r:DIRECTED]->(b:base {entity_id: pair.tgt})
RETURN pair.src AS source_id, pair.tgt AS target_id, properties(r) AS props
```
Construct GraphEdge for each result. Return dict mapping (source_id, target_id) -> GraphEdge.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-GRAPH-01 | Tampering | graph.py Cypher injection via node_id | mitigate | All node/entity IDs passed via {params: json.dumps({...})} as $1::agtype — PostgreSQL AGE treats them as agtype map values, NOT as Cypher syntax. json.dumps() escapes all special characters. Never is any user value interpolated into Cypher string. |
| T-02-04-GRAPH-02 | Tampering | graph.py dollar-quote collision with Cypher content | mitigate | _dollar_quote() uses iterative unique tag generation (AGE1, AGE2, ...) checking for collision with the content being quoted. If content contains the tag, next counter is tried. |
| T-02-04-GRAPH-03 | Elevation of Privilege | graph.py write operations | mitigate | All queries use conn.fetch() (read-only). _query() helper exclusively uses fetch(). No INSERT/UPDATE/DELETE/CREATE path exists. server_settings={'default_transaction_read_only': 'on'} at pool level provides defense-in-depth (D-15). |
| T-02-04-GRAPH-04 | Information Disclosure | graph.py graph name in error messages | accept | Graph name may appear in error messages — it is a configuration identifier, not a secret. No passwords, connection strings, or raw data leak through error paths. |
| T-02-04-GRAPH-05 | Denial of Service | graph.py unbounded graph traversal | mitigate | get_nodes_batch accepts explicit list (caller controls size). get_node_edges returns neighbor IDs only (lightweight). Depth not traversed beyond 1 hop. No recursive/gremlin-style unbounded traversal. |
| T-02-04-GRAPH-06 | Tampering | graph.py agtype parsing of ::vertex/::edge suffixes | mitigate | _parse_agtype() splits on "::" and json.loads the content portion. On JSONDecodeError, returns None — never crashes or exposes raw data. Graceful degradation for malformed AGE output. |
