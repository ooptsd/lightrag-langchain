Imports: import asyncpg (module-level, not TYPE_CHECKING — used at runtime). from lightrag_langchain.config import settings. from lightrag_langchain.data.models import EntityRecord, RelationshipRecord, ChunkRecord. from lightrag_langchain.data.pool import acquire_with_retry, pool as _default_pool.

Define PGVectorStore class:

Constructor __init__(self, pool: asyncpg.Pool | None = None, workspace: str | None = None, table_prefix: str = "LIGHTRAG_VDB"):
- self._pool = pool (D-07: dependency injection — if None, falls back to module-level pool via _default_pool)
- self._workspace = workspace if workspace is not None else settings.pg.workspace (D-05: single workspace, default "default")
- self._table_prefix = table_prefix
- self._tables: dict[str, str] | None = None  # Lazy-discovered table name cache
- self._cosine_threshold = settings.query_params.cosine_threshold
- self._default_top_k = settings.query_params.top_k

Property self.pool: returns self._pool if not None, else _default_pool. Raises RuntimeError if both are None (pool not initialized).

Method async _ensure_tables(self) -> dict[str, str]:
- If self._tables is not None, return cached result.
- Query information_schema.tables WHERE table_schema='public' AND table_name LIKE $1 ORDER BY table_name. Use self._table_prefix + "%" as the LIKE pattern.
- Parse results into namespaces: ENTITY, RELATION, CHUNKS.
  - ENTITY matches: table_name startswith "{prefix}_ENTITY" (exact match or "{prefix}_ENTITY_{suffix}")
  - RELATION matches: table_name startswith "{prefix}_RELATION"
  - CHUNKS matches: table_name startswith "{prefix}_CHUNKS"
- For each namespace with >1 match: raise RuntimeError with message listing all found variants and instructing user to set PG_TABLE_SUFFIX in .env (D-13).
- For each namespace with 0 matches: raise RuntimeError with message "No {namespace} table found with prefix '{prefix}'. Is the LightRAG database initialized?"
- For each namespace with exactly 1 match: store the full table_name in result dict.
- Cache result in self._tables and return.

Method async _vector_search(self, table_name: str, embedding: list[float], top_k: int, select_clause: str) -> list[dict[str, Any]]:
- Compute closer_than = 1.0 - self._cosine_threshold (cosine distance = 1 - cosine similarity)
- Build SQL: f"{select_clause} FROM {table_name} WHERE workspace = $1 AND content_vector <=> $4::vector < $2 ORDER BY content_vector <=> $4::vector LIMIT $3"
  - $1 = workspace string
  - $2 = closer_than (float)
  - $3 = top_k (int)
  - $4 = embedding (list[float] → pgvector vector via register_vector codec)
- Use async with acquire_with_retry(self.pool) as conn: rows = await conn.fetch(sql, self._workspace, closer_than, top_k, embedding)
- Return [dict(row) for row in rows]

Method async search_entities(self, embedding: list[float], top_k: int | None = None) -> list[EntityRecord]:
- top_k = top_k if top_k is not None else self._default_top_k
- await self._ensure_tables()
- select_clause = """SELECT entity_name, content, id AS source_id, COALESCE(file_path, '') AS file_path, EXTRACT(EPOCH FROM create_time)::BIGINT AS created_at"""
- rows = await self._vector_search(table_name, embedding, top_k, select_clause)
- Return [EntityRecord(**row) for row in rows]

Method async search_relationships(self, embedding: list[float], top_k: int | None = None) -> list[RelationshipRecord]:
- Same top_k default logic
- select_clause = """SELECT source_id AS src_id, target_id AS tgt_id, content, NULL::text AS keywords, NULL::float8 AS weight, EXTRACT(EPOCH FROM create_time)::BIGINT AS created_at"""
  - keywords and weight are SELECT NULL because VDB_RELATION DDL does not contain these columns (per RESEARCH.md A1 assumption and Open Question 1). They are returned as None in RelationshipRecord. Real keywords/weight values come from AGE graph edges (Plan 02-04 PGGraphStore).
- Return [RelationshipRecord(**row) for row in rows]

Method async search_chunks(self, embedding: list[float], top_k: int | None = None) -> list[ChunkRecord]:
- Same top_k default logic
- select_clause = """SELECT id AS chunk_id, content, full_doc_id, chunk_order_index, COALESCE(file_path, '') AS file_path"""
- Return [ChunkRecord(**row) for row in rows]

All three search methods: use self._tables[namespace] for table_name after _ensure_tables().

All SQL uses conn.fetch() exclusively — never conn.execute() (D-15 read-only). The _vector_search method uses parameterized $1/$2/$3/$4 placeholders — never f-string interpolation for values (anti-pattern from RESEARCH.md).

Table names in f-string SQL are safe because they come from information_schema (system catalog, not user input) and are validated by _ensure_tables() which only allows tables matching the LIGHTRAG_VDB_* prefix pattern. This is the whitelist approach documented in RESEARCH.md security section.

Do NOT import or use huggingface, sentence-transformers, or any embedding library. PGVectorStore receives pre-computed vectors (list[float]) as input (D-10).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller → PGVectorStore.search_*() | Embedding vector (list[float]) crosses boundary from Phase 3 LLM layer |
| PGVectorStore → PostgreSQL | Parameterized SQL with $1..$4 placeholders; table names from information_schema (system catalog whitelist) |
| PGVectorStore → asyncpg Connection | conn.fetch() only — no conn.execute() path exists |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-STORE-01 | Tampering | store.py table_name in f-string SQL | mitigate | Table names sourced exclusively from information_schema.tables query results matching LIGHTRAG_VDB_% prefix — a system catalog whitelist. _ensure_tables() validates exactly 1 match per namespace before any query executes. No user input reaches table_name. |
| T-02-03-STORE-02 | Tampering | store.py vector injection (malformed embedding) | mitigate | Embedding is list[float] validated by Python type system. content_vector <=> $4::vector ensures PostgreSQL treats it as pgvector type. TOP_K + COSINE_THRESHOLD double-limit result set size. command_timeout=30s caps query duration. |
| T-02-03-STORE-03 | Elevation of Privilege | store.py write operations | mitigate | All queries use conn.fetch() (returns rows, no side effects). No conn.execute() call exists in any code path. server_settings={'default_transaction_read_only': 'on'} at pool level provides defense-in-depth (D-15). |
| T-02-03-STORE-04 | Information Disclosure | store.py error messages | mitigate | Error messages reference table prefixes and namespace names only — never expose connection strings, passwords, or raw query data. PG_PASSWORD accessed exclusively via SecretStr.get_secret_value() in pool.py (Plan 02-02). |
| T-02-03-STORE-05 | Denial of Service | store.py unbounded vector search | mitigate | top_k bounded by caller (default 40 from settings). cosine_threshold (default 0.2) filters low-similarity results server-side. command_timeout (30s) limits individual query execution time. Pool max_size=10 prevents connection exhaustion. |
| T-02-03-STORE-06 | Denial of Service | store.py information_schema query | accept | Single metadata query per store instance (cached via _tables). Low cost — information_schema.tables is a system view. No attacker-controlled input. |
</threat_model>
